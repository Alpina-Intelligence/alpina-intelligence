# Building a Modern Blog with TanStack Start and MDX

In this comprehensive guide, we'll explore how to build a modern, performant blog using some of the latest web technologies. We'll be using **TanStack Start** for our framework, **MDX** for content authoring, and **Shiki** for beautiful syntax highlighting.

## Why These Technologies?

Before diving into the implementation, let's understand why these technologies work so well together:

- **TanStack Start**: A full-stack React framework with excellent developer experience
- **MDX**: Allows you to write JSX directly in your markdown files
- **Shiki**: Provides VS Code-quality syntax highlighting with zero runtime overhead

## Getting Started

### Prerequisites

Make sure you have the following installed:

- Node.js 18 or higher
- Bun package manager
- Basic knowledge of React and TypeScript

### Installation

First, let's install the required dependencies:

```bash
bun create @tanstack/start my-blog
cd my-blog
bun install
```

Next, add the MDX and Shiki packages:

```bash
bun add @mdx-js/rollup @mdx-js/react shiki
bun add remark-gfm rehype-slug rehype-autolink-headings
```

## Creating the Markdown Renderer

The heart of our blog is the markdown renderer. Here's a simplified version:

```typescript
import { type ComponentProps, type ReactNode } from 'react'
import { CodeBlock } from './code-block'

export function MarkdownRenderer({ children }: { children: ReactNode }) {
  return (
    <article className="prose prose-neutral dark:prose-invert max-w-none">
      {children}
    </article>
  )
}

export const mdxComponents = {
  code: ({ children, className }: ComponentProps<'code'>) => {
    const isInline = !className
    if (isInline) return <code>{children}</code>
    return <CodeBlock className={className}>{String(children)}</CodeBlock>
  },
  // ... other components
}
```

## Implementing Syntax Highlighting with Shiki

Shiki provides beautiful, accurate syntax highlighting. Here's how to integrate it:

```typescript
import { codeToHtml } from 'shiki'

export async function highlightCode(code: string, lang: string) {
  return await codeToHtml(code.trim(), {
    lang,
    theme: 'github-dark',
  })
}
```

> **Note**: Shiki uses TextMate grammars and VS Code themes, ensuring your code looks exactly like it does in your editor.
